### 4.2 ECOSISTEMANET — Infraestructura para Ecosistemas Complejos
**Rol:** Cofundador (diciembre 2025 - presente)

#### Identidad
EcosistemaNet es una **infraestructura digital modular** diseñada para representar, analizar, coordinar y activar ecosistemas de innovación y emprendimiento. No vende software genérico — vende **soluciones a problemas específicos** donde el cliente compra **claridad para reducir incertidumbre en contextos complejos.**

> EcosistemaNet no es solo software. Es una infraestructura modular para reducir incertidumbre, coordinar decisiones y activar ecosistemas de innovación y emprendimiento.

#### Las 4 funciones estratégicas
1. **Hacer visible lo difuso:** identifica actores, vínculos, capacidades, dinámicas y vacíos
2. **Transformar datos dispersos en inteligencia accionable:** formularios, conversaciones, evaluaciones, métricas → insumos para decisión
3. **Orquestar decisiones colectivas:** gobernanza, priorización, evaluación, trazabilidad
4. **Reducir incertidumbre para actuar mejor:** esta es la formulación más potente

#### Problema que resuelve
- Fragmentación de información (planillas, PDFs, correos, conocimiento informal)
- Baja visibilidad sistémica del ecosistema
- Débil capacidad de diagnóstico — decisiones con información parcial o anecdótica
- Procesos de decisión poco trazables
- Evaluación inconsistente de proyectos y postulaciones
- Falta de seguimiento operativo post-decisión

#### Los 8 módulos
| Módulo | Propósito | Claridad que entrega |
|---|---|---|
| **Atlas** | Mapear, visualizar y gestionar ecosistemas complejos a través de actores y relaciones | Claridad estructural |
| **Pulse** | Levantar información crítica y construir diagnósticos estratégicos | Claridad diagnóstica |
| **Focus** | Transformar conversaciones en información estructurada mediante análisis automatizado | Claridad conversacional |
| **Vantage** | Evaluar proyectos con rúbricas técnicas e IA contextualizada | Claridad de evaluación |
| **Nexus** | Gestionar votaciones y toma de decisiones para grupos cerrados con trazabilidad | Claridad de decisión |
| **Metron** | Medir y gestionar uso de tiempo y recursos en iniciativas | Claridad operativa |
| **Skopia** | Explorar, mapear y diagnosticar el talento emprendedor del ecosistema | Claridad de talento |
| **Agora** | Gestionar eventos, participación y datos de interacción | Claridad de participación |

#### Flujo principal del sistema
Pulse diagnostica → Atlas revela estructura → Focus convierte conversación en evidencia → Vantage evalúa → Nexus decide → Metron ejecuta → Agora captura interacción → todo retroalimenta el modelo del ecosistema.

#### Flujos de uso concretos
- **Diagnóstico del ecosistema:** Pulse recoge datos → Focus captura cualitativo → Atlas estructura el mapa
- **Evaluación de convocatorias:** Pulse recibe postulaciones → Vantage evalúa → Nexus decide en colegiado → Metron acompaña ejecución
- **Gobernanza y participación:** Atlas identifica actores → Nexus organiza decisión → Focus captura acuerdos → Metron sigue compromisos
- **Activación del ecosistema:** Agora gestiona eventos → Focus convierte interacción en conocimiento → Atlas incorpora señales

#### Tipos de clientes
- **Ancla:** gobiernos regionales, programas públicos, universidades, corporaciones de innovación
- **Modulares:** incubadoras, aceleradoras, hubs, consultoras, redes de emprendimiento
- **Operativos:** organizaciones que ejecutan convocatorias, administran comités, necesitan trazabilidad

#### Modelo de negocio
- Implementación inicial (configuración, carga, modelado, adaptación)
- Suscripción SaaS (acceso por usuario u organización, módulos base)
- Servicios premium (consultoría analítica, operación asistida, acompañamiento de gobernanza)
- Consumo variable — especialmente Focus y Vantage (créditos por volumen de análisis)

**Enfoque comercial:** entrada por dolor específico → expansión modular por valor entregado. El cliente no necesita todo desde el inicio.

#### Stack tecnológico
- Frontend modular (analítica, administración, evaluación, gobernanza, operación)
- Backend de negocio (workflows, reglas, permisos, scoring, trazabilidad)
- Capa de datos (base relacional + almacenamiento documental + búsqueda semántica)
- Capa de IA (embeddings, RAG, transcripción, clasificación, scoring contextual, síntesis)
- Infraestructura multi-tenant con seguridad estricta y parametrización por cliente

#### IA en EcosistemaNet
- Focus: transcripción y síntesis de conversaciones
- Vantage: scoring contextual combinando rúbricas técnicas con criterio experto codificado
- Búsqueda semántica sobre documentos y bases de datos
- Detección de patrones y recomendaciones automáticas
- Clasificación de actores y startups
- Extracción de compromisos y temas de reuniones

#### Diferenciación competitiva
- Modela complejidad real (actores, relaciones, decisiones, conversaciones, talento, ejecución)
- Conecta capas normalmente separadas en un solo sistema
- Propuesta de claridad, no de feature list
- Mezcla software con método (especialmente Atlas y Vantage)
- Barreras de entrada: metodológica, de datos, funcional e institucional

#### Evolución roadmap
- **Etapa 1:** plataforma estructurada (captura, organiza, evalúa, decide, mide)
- **Etapa 2:** plataforma inteligente (interpreta, recomienda, alerta, sintetiza con IA)
- **Etapa 3:** plataforma semiautónoma (detecta señales, propone acciones, activa workflows, aprende)

**Módulos futuros en radar:** Observatorio, Recomendador, Portafolio, Impacto longitudinal, Copiloto EcosistemaNet.
